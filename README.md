Backend service for the **Permatatex IT Inventory System** (Enterprise scale), developed for **PT. Permata Anugrah Kusuma** (Garment/Textile industry). This system manages complex workflows from raw materials to finished goods, work order routing, and partner/client integration.

---

## 🛠️ Technology Stack

*   **Language:** Golang v1.26.0
*   **Web Framework:** Gin (`github.com/gin-gonic/gin`)
*   **Database:** PostgreSQL
*   **Database Pool/Driver:** `pgx` v5 (`github.com/jackc/pgx/v5`)
*   **Query Builder:** `sqlc` (Pure SQL compilation to type-safe Go, NO ORM/GORM)
*   **Database Migrations:** `golang-migrate` (.sql up/down scripts)
*   **Configuration Manager:** Viper (`github.com/spf13/viper`) for `.env` loading
*   **Authentication & Security:** JWT (JSON Web Tokens) & Cloudflare Turnstile CAPTCHA validation
*   **Structured Logging:** Standard Library `log/slog` (Structured JSON output)
*   **API Documentation:** Swagger (`swaggo/swag`)
*   **Live Reload & Task Runner:** Air (`github.com/air-verse/air`) & GNU Make

## 🏛️ Clean Architecture Design

This project strictly adheres to Concern Separation based on the **Delivery-Model-Entity** pattern:

```text
permatatex-project/backend/
├── cmd/
│   ├── web/               # Server initialization, router setup, dependency injection (main.go)
│   └── seeder/            # Database seeder scripts for mock data
├── db/
│   ├── migrations/        # Database schema migrations (up/down .sql files)
│   └── query/             # Pure SQL statements compiled by sqlc
├── internal/
│   ├── config/            # Viper configurator and pgx connection pool initialization
│   ├── delivery/
│   │   └── http/          # Gin Route Handlers & Middlewares (handles REST, JSON binding, validation)
│   ├── usecase/           # Pure business logic implementation (orchestrates repos and entities)
│   ├── repository/        # Database query calls (calls sqlc-generated routines)
│   ├── gateway/           # Third-party integrations (Cloudflare Turnstile HTTP Gateway)
│   ├── entity/            # Database schema representations & interfaces generated by sqlc
│   └── model/             # DTOs (Data Transfer Objects) for requests and responses
├── pkg/                   # Reusable library packages (response wrappers, logger, JWT token generator)
└── docs/                  # Swagger spec documentation files
```

## 🚀 Getting Started

### Prerequisites
*   Go v1.26 or higher
*   Docker & Docker Compose
*   PostgreSQL v15+ (if running database locally outside Docker)

### Configuration Setup
1. Copy the environment configuration template:
   ```bash
   cp .env.example .env
   ```
2. Modify the values in `.env` to match your local setup.

### Database Setup & Migration
To run the database migrations and apply changes to PostgreSQL:
```bash
# Run migrations up (locally)
make migrate-up

# Or run migrations using Docker network
make migrate-up-docker
```

To rollback database changes:
```bash
make migrate-down
```

### Seeding Dummy Data
To populate the database with system roles, permissions, master sizes, colors, and initial users:
```bash
make seed
```

### Running the Server
Start the development server with live-reloading (using Air):
```bash
make dev
```
The server will start on port `8080` (or the port defined in `.env`).

## 🛠️ Commands Reference (Makefile)

| Command | Action |
| :--- | :--- |
| `make dev` | Starts local development server using Air live-reload |
| `make seed` | Runs database seeder (`cmd/seeder/main.go`) |
| `make db-gen` | Re-compiles SQL queries in `db/query` using `sqlc` |
| `make migrate-up` | Runs golang-migrate to sync PostgreSQL schema to the latest version |
| `make migrate-down` | Rollback the database by 1 migration step |
| `make swag` | Re-generates Swagger API documentation |
| `make lint` | Audits code quality using `golangci-lint` |
| `make lint-fix` | Auto-corrects lint issues where possible |
| `make docker-up` | Launches the stack (`db` and `app`) via docker compose |
| `make docker-down` | Tear down the running Docker containers |
